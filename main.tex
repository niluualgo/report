\chapter*{Abstract}

In high-density urban environments, losing an item is often a chaotic experience defined by panic and a lack of actionable data. Current methods rely on an "illusion of recovery," where people falsely believe lost items are easily found through traditional means, despite evidence to the contrary. 

This project proposes \textbf{"Reunite,"} a mobile application that transforms the lost-and-found process from a passive "waiting game" into an active, algorithmically secured recovery mission. The key innovation of this system is the implementation of a "Trust Network" which prevents scams and ensures secure handovers through asymmetric privacy and verification protocols. By utilizing geospatial scoring, real-time data synchronization, and automated greedy clustering for map visualization, the application streamlines recovery while protecting user identity and property.

\chapter{INTRODUCTION}

\section{General Background}
The project addresses the experience of losing items in busy urban settings. This experience is specifically defined by panic and a complete lack of actionable data to track the item. Society currently suffers from an "illusion of recovery," convincing ourselves that lost items will be found, despite vast evidence to the contrary. In reality, we rely on outdated, traditional methods to find these items, which are often ineffective, disjointed, and insecure.

\section{Objective}
The primary objective of this project is to create an active, algorithmically secured recovery platform. By replacing vague physical searching with dynamic spatial mapping and enforcing asymmetric privacy during handovers, the system aims to establish a secure community-driven "Trust Network" that dramatically increases the recovery rate of lost urban property.

\section{Problem Statement}
The traditional ecosystem of item recovery presents severe functional flaws. Scammers frequently use publicly posted photos of found items to fake ownership ("The Claim Scam"). Vague descriptions leave searchers with "Spatial Blindness," and overlapping digital pins create map clutter. Furthermore, old data stays up indefinitely, creating "Ghost Data" that reduces system efficiency.

\section{Scope}
The scope of Project Spear encompasses the development of a cross-platform mobile application utilizing Flutter, Supabase, and Mapbox to facilitate secure lost-and-found property recovery. It is designed for high-density urban areas, college campuses, and transit hubs. The system handles real-time geospatial tracking, cryptographic identity verification, and map decluttering, but currently excludes hardware tracking tags (like AirTags) and limits matching purely to user-submitted database entries.

\chapter{SYSTEM DEVELOPMENT}

\section{Existing system}
The existing methods for recovering lost items are fundamentally flawed across several domains. Users must rely on manual keyword searches or endless scrolling through social media feeds and forums. Organization is typically presented as an unsorted, unverified list. Furthermore, the physical return of items is completely unverified and carries inherent physical and property risks.

\subsection{Disadvantages}
\begin{itemize}
    \item High vulnerability to false claims and scammers stealing property.
    \item Visual clutter and data fatigue due to non-clustered, localized pins.
    \item Persistent "Ghost Data" from unremoved old posts that obscure recent lost items.
\end{itemize}

\section{Proposed System}
To counter the limitations of the existing systems, the "Reunite" platform introduces advanced mechanisms including Asymmetric Privacy, an Algorithmic Scoring Engine, $O(N)$ Greedy Clustering for maps, and Cryptographic QR Verification for secure handovers.

\subsection{Advantages}
\begin{itemize}
    \item \textbf{Scam Immunity:} The Asymmetric Privacy engine eliminates visual bait for scammers.
    \item \textbf{UX Clarity:} The integrated Greedy Clustering ensures maps remain highly usable.
    \item \textbf{High Performance:} Lightweight algorithms ensure the app remains snappy.
    \item \textbf{Real-Time Agility:} WebSockets deliver millisecond updates, effectively eliminating "Ghost Data."
\end{itemize}

\section{Modules}

\subsection{Matching and Scoring Engine}
The backend "brain" that mathematically calculates match probabilities by weighing proximity, time freshness, and attribute similarity.

\subsection{Asymmetric Privacy Controller}
Prevents fraudulent claims by actively obfuscating item details (like blurring photos) until a user proves ownership.

\subsection{Map Visualization \& Clustering}
Uses MapBox and an $O(N)$ greedy clustering algorithm to manage high-density item pins and suggest secure "Safe Zones" for meeting.

\subsection{Real-Time Data Sync}
Utilizes Supabase WebSockets to eliminate "ghost data" by pushing instant data updates to all connected users.

\section{System analysis}

\subsection{Testing analysis}
Software testing is a critical phase of the system development life cycle to ensure the Reunite platform operates without failure under various conditions. Testing was conducted rigorously to identify and resolve defects in the Flutter UI and PostgreSQL backend.

\subsection{Unit Testing}
Unit testing involved isolating and testing individual modules. The $mScore$ mathematical logic was tested with dummy variables to ensure accurate probability outputs. The Mapbox clustering algorithm was also unit-tested to verify that it correctly grouped nearby coordinate arrays without crashing.

\subsection{System Testing}
System testing evaluated the fully integrated application. This included testing the real-time WebSocket connections by simulating simultaneous posts from multiple users to ensure data synchronized across devices without manual refreshing. The QR verification protocol was also tested end-to-end to confirm cryptographic handshake success.

\section{Performance Analysis}
The platform demonstrated exceptional operational stability. By utilizing greedy clustering algorithms ($O(N)$ complexity), the application minimized frame drops, successfully maintaining 60 frames per second during heavy map zooming and panning. Real-time updates pushed from Supabase resolved in under 200 milliseconds, ensuring users never viewed obsolete "ghost data".

\chapter{CONCLUSION}
The development of the "Reunite" application marks a significant paradigm shift in urban property recovery. It successfully transforms the lost-and-found process from a passive, panic-inducing "waiting game" into an active, algorithmically directed recovery mission. By prioritizing security, the application replaces vague social posts with a rigorous, mathematically secured system that prevents fraud and protects user identities. Ultimately, it successfully establishes a secure, real-time "Trust Network" capable of serving the community.

\section{Future Scope}
Several enhancements are planned for future iterations:
\begin{itemize}
    \item \textbf{AI Image Matching:} Integrating advanced computer vision models to automatically match found items with lost item descriptions based on uploaded images.
    \item \textbf{Wider Geographic Scaling:} Expanding the backend infrastructure to handle multi-city geospatial clustering.
    \item \textbf{Cross-Platform Web Client:} Developing a unified web portal to allow administrative access for campus security and local authorities.
\end{itemize}
